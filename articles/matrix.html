Meta AI Researchers Introduce Matrix — Ray-Native, Decentralized Framework for Multi-Agent Synthetic Data Generation

How Matrix removes central bottlenecks to scale synthetic data generation across thousands of GPUs.
▶ Live Demo View Repo
Published: Nov 2025
—
Category: Research • Systems
Performance
6.8× tokens/s • 15.4× tool-use
Scale: thousands of GPUs

Modern AI models need huge amounts of synthetic (AI-generated) data — like conversations, multi-step reasoning traces, and tool-use logs. Producing this data at scale typically requires many cooperating AI agents. Most existing systems use a central controller to orchestrate agents, which creates a major scalability bottleneck: when thousands of tasks run concurrently, the controller throttles throughput and wastes GPU cycles.

Matrix: Decentralized Orchestration

Matrix is a Ray-native decentralized framework introduced to remove the central controller and let tasks flow through stateless agents. Instead of a global scheduler, every logical task becomes a self-contained message (an orchestrator message) that carries state, history, and intermediate results.

Orchestrator message = task state + conversation history + results so far.

Message contents

  1. The current task state
  2. Conversation / interaction history
  3. Intermediate outputs and results

How work flows

Stateless worker agents read orchestrator messages from distributed queues, perform their assigned transformations, update the message, and route it to the next agent. Because each task advances independently, the system avoids global waits and achieves massive parallelism.

Matrix runs on Ray and integrates with systems like vLLM, SGLang, SLURM, monitoring tools such as Grafana, and common container/storage tooling. For very long conversations, Matrix keeps compact references to older segments in shared storage instead of loading full histories, which reduces memory and network pressure.

Performance results

Benchmarks reported by the authors include:

  1. Collaborative Reasoning — 6.8× more tokens per second
  2. NaturalReasoning Web Data — 2.1× higher throughput
  3. Tau2 Bench (Tool Use) — 15.4× faster execution

Importantly, these speedups come without a measurable drop in generation quality, showing that improved system design — not only model size — can unlock orders of magnitude more synthetic data generation capacity.

Main idea

Matrix shows that decentralizing orchestration and using message-driven workflows transforms AI pipelines into fast, flexible, and fault-tolerant distributed systems. For teams building multi-agent synthetic data pipelines, this is a practical architecture to scale training data generation across thousands of GPUs.
